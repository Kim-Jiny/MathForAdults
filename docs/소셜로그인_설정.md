# 소셜 로그인 설정 가이드 (Apple · Google)

선택적 로그인(익명 우선) + 진도 클라우드 동기화를 위한 **콘솔/네이티브 설정**.
코드는 이미 구현돼 있고, 아래 발급값만 채우면 실제 로그인이 동작합니다.

- 서버 API: `https://duo.jiny.shop/api/mathforadults`
  - `POST /auth/social` `{ provider, idToken }` → 검증 후 자체 JWT 발급
  - `GET/PUT /progress` (Bearer) → 진도 동기화
- 토큰 검증: 서버 `server/src/services/mfaAuth.ts` (Google=google-auth-library, Apple=Apple JWKS)

---

## 1. 서버 환경변수 (Minigame `server/.env`)

```
JWT_SECRET=<이미 운영중인 값 그대로>
# Google: 검증 시 허용할 client_id(audience) 목록 — 콤마 구분
#   (Android client id, iOS client id, Web client id 모두 넣어두면 안전)
MFA_GOOGLE_CLIENT_IDS=xxxxx-android.apps.googleusercontent.com,xxxxx-ios.apps.googleusercontent.com,xxxxx-web.apps.googleusercontent.com
# Apple: identityToken 의 audience = iOS 번들 ID (필요시 Service ID 추가)
MFA_APPLE_BUNDLE_IDS=com.jiny.mathForAdults
```

설정 후 서버 재빌드·재시작하면 `mfa_users`·`mfa_progress` 테이블이 자동 생성됩니다.

---

## 2. Google 로그인

### Google Cloud Console
1. 프로젝트 → API 및 서비스 → 사용자 인증 정보(OAuth 동의 화면 먼저 구성).
2. OAuth 클라이언트 ID 발급:
   - **Android**: 패키지명 `com.jiny.math_for_adults` + 서명 SHA-1 (아래 명령으로 확인)
   - **iOS**: 번들 ID `com.jiny.mathForAdults` (※ iOS는 카멜케이스 — Android와 다름)
   - **웹 애플리케이션**: 서버 검증용 — 이 **웹 클라이언트 ID** 를 앱의 `serverClientId` 로 사용
3. 위에서 받은 client_id 들을 서버 `MFA_GOOGLE_CLIENT_IDS` 에 모두 등록.

SHA-1 확인 (릴리스 + 디버그):
```bash
# 릴리스(업로드 키)
keytool -list -v -keystore android/upload-keystore.jks -alias upload -storepass <키스토어_비밀번호> | grep SHA1
# 디버그
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android | grep SHA1
```

### 앱 빌드 시 주입
```bash
flutter build appbundle --release --dart-define=GOOGLE_SERVER_CLIENT_ID=<웹 클라이언트 ID>.apps.googleusercontent.com
```

### iOS 추가 (google_sign_in)
- `ios/Runner/Info.plist` 에 iOS 클라이언트 ID 와 역방향 URL 스킴 추가:
```xml
<key>GIDClientID</key>
<string>xxxxx-ios.apps.googleusercontent.com</string>
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleURLSchemes</key>
    <array><string>com.googleusercontent.apps.xxxxx-ios</string></array>
  </dict>
</array>
```

## 3. Apple 로그인 (iOS 우선)

### Xcode
1. `open ios/Runner.xcworkspace` → Runner 타깃 → **Signing & Capabilities** → `+ Capability` → **Sign in with Apple** 추가.
   (이러면 `Runner.entitlements` 에 `com.apple.developer.applesignin` 이 추가됨)
2. Apple Developer → Identifiers → App ID(`com.jiny.mathForAdults`)에서 **Sign In with Apple** 체크.

### 서버
- `MFA_APPLE_BUNDLE_IDS` 에 번들 ID(`com.jiny.mathForAdults`) 등록.
- Android에서도 Apple 로그인을 지원하려면 Service ID + 웹 리다이렉트 흐름이 필요(별도 설정). iOS 전용으로 두는 것을 권장.

---

## 4. 동작/동기화 요약
- 설정 → 계정 → Apple/Google 로그인.
- 로그인 직후: 클라우드 진도를 내려받아 **로컬과 병합**(누적값은 큰 쪽, 출석·오답은 합집합) 후 다시 업로드.
- "지금 동기화" 버튼으로 수동 병합·업로드. 로그아웃 시 로컬 데이터는 그대로 유지.
- 미설정 상태에서 로그인 시도 시: 서버 검증 실패로 "로그인에 실패했어요" 표시(앱은 익명으로 정상 사용).

---

## 5. 미설정이어도 안전
- 로그인은 **선택**입니다. 콘솔 설정 전에도 앱은 익명(로컬)으로 완전히 동작합니다.
- `GOOGLE_SERVER_CLIENT_ID` 미주입 / 서버 env 미설정이면 로그인만 실패하고 나머지 기능엔 영향 없음.
