# 성인의 수학 — 제품 기획서 (as-built)

> 20~40대 성인이 취미처럼 수학을 다시 푸는 무압박 학습 앱.
> 쉬운 문제부터 시작해 결국 수능 수학까지 도전. 단원 체계는 수능 커리큘럼 그대로.

- 문서 버전: v1.0 (현재 구현 반영)
- 플랫폼: Flutter (iOS / Android, web도 빌드 가능)
- 상태: 더미 데이터 기반 동작 (서버 연동 전)
- 수식: LaTeX 렌더링 지원(flutter_math_fork) — 더미는 유니코드 사용

---

## 1. 제품 개요

### 1.1 한 줄 정의
수능 수학 커리큘럼을 정확히 따르되, 성인이 부담 없이 "오늘 한 문제" 풀게 만드는 취미형 수학 루틴 앱.

### 1.2 디자인 방향
- 키즈앱처럼 유치하지 않게, 인강/문제집처럼 딱딱하지 않게.
- 성인이 써도 부끄럽지 않은 **차분하고 깔끔한** 톤.
- "공부해야 한다"가 아니라 **"오늘 한 문제 풀어볼까?"**.
- 카드 기반 UI, 넉넉한 여백, 명확한 텍스트 계층.
- 문제 풀이 화면의 **가독성이 최우선**. 장식 최소화.
- 좌표/곡선 등 **은은한 수학적 분위기**를 배경으로 약하게 활용.

### 1.3 타겟
재도전형 직장인(주력) · 자녀 동반 학습 부모 · 검정고시/편입 등 시험 준비 성인.
공통 특성: 시간이 파편적, 압박에 민감, 자기주도적.

## 2. 정보 구조

계층: **과목(Subject) → 단원(chapter) → 세부 단원(lesson) → 난이도(Difficulty) → 문제(Problem)**

- 세부 단원(lesson)은 문제 데이터의 `lesson` 필드에서 동적으로 수집된다 → 문제만 추가하면 화면에 반영.
- 과목 트리는 7과목 전부 탑재. 문제는 점진 탑재(없는 단원은 "준비 중").

### 2.1 과목·단원 (실제 단원명 유지)
- **중학 수학**: 수와 연산 · 문자와 식 · 방정식과 부등식 · 함수 · 도형 · 확률과 통계
- **고등 공통수학**: 다항식 · 방정식과 부등식 · 도형의 방정식 · 집합과 명제 · 함수 · 경우의 수
- **수학Ⅰ**: 지수함수와 로그함수 · 삼각함수 · 수열
- **수학Ⅱ**: 함수의 극한과 연속 · 미분 · 적분
- **확률과 통계**: 경우의 수 · 확률 · 통계
- **미적분**: 수열의 극한 · 미분법 · 적분법
- **기하**: 이차곡선 · 평면벡터 · 공간도형과 공간좌표

### 2.2 난이도 6단계
개념 확인 → 기본 유형 → 대표 유형 → 응용 유형 → 수능 기초 → 수능 실전
(수능 랜덤은 2점=개념·기본, 3점=대표·응용, 4점=수능기초·실전으로 매핑)

### 2.3 콘텐츠 목표 (최종 규모)
- **목표 밀도: 세부 단원(소단원) × 난이도 단계당 약 50문제.**
  - 예) 수학Ⅰ > 수열 > 등차수열 의 "대표 유형" 한 칸에 ~50문제.
  - 중학 수학은 '수능 실전' 단계 제외(그 외 5단계).
- 추산 총량: 세부 단원 약 99개 × 평균 5~6단계 × 50 ≈ **2.5만~3만 문제** (장기 목표).
- **점진 충전 전략**: 한 번에 다 만들지 않는다. 기능을 먼저 단단히 완성하고, 콘텐츠는 단원/난이도 단위로 워크플로우를 반복해 채운다.
  - 파이프라인: 생성(sonnet) → 1차 독립 재계산 검증 → 2차 단원별 전수 재검증 → `merge_generated.py` 병합(중복·결함 제거) → `build_index.dart` 인덱스 재생성 → `flutter test` 무결성.
  - 자동 생성 id `gen_<slug>_<n>`, 손수 추가 `hand_*`.
- **현재 상태(시드)**: 과목당 ~200, 총 ~1,422문제(검증 완료). 이는 구조 검증용 시드이며 위 목표의 일부.
- 용량 영향: 3만 문제도 텍스트 기준 ~수 MB로 앱 크기 문제 없음. 규모가 더 커지면 단원별 lazy 로딩(이미 적용)에 더해 서버 다운로드형 문제팩으로 확장 가능.

## 3. 화면 (하단 탭 5개)

탭: **홈 · 단원 · 다시풀기 · 기록 · 설정**

| 화면 | 핵심 |
|---|---|
| 홈 | 인사+날짜, 오늘의 문제(수학 배경 카드), 이어 풀던 단원, 수능 랜덤(2/3/4점), 다시 풀 문제, 이번 주 기록 |
| 단원 | 과목 카드(진행률) → 단원 목록(추천 난이도) → 세부 단원 → 난이도 바텀시트 → 풀이 |
| 문제 풀이 | 위치·난이도뱃지·진행률, 큰 본문, 큰 4지선다, 선택 즉시 채점, 해설 패널 |
| 정답/해설 | 정답 여부(부드러운 칭찬/부담 없는 문구), 정답표시, 짧은 해설, 상세 해설(접기), 다시풀문제 저장, 비슷한 문제 |
| 다시풀기 | 틀린 문제 목록, 과목/난이도 필터, 최근일·정답률·재도전, "오늘 다시 풀 문제" |
| 기록 | 푼 문제·정답률·연속·이번주·약한 단원, 과목별 진행률, 최근 풀이 기록 |
| 설정 | 알림, 하루 목표(1/3/5/자유), 테마(시스템/라이트/다크), 문의, 앱 정보 |

### 3.1 메인 흐름
```
홈 "오늘 한 문제" → 풀기            (가장 빠른 진입)
단원 → 과목 → 단원 → 세부단원 → 난이도 → 풀이   (탐색 진입)
풀이 → 즉시 채점 → 해설 → 다음 문제 → 세션 결과
오답 → 자동으로 '다시풀기'에 누적
```

## 4. 데이터 / 상태

### 4.1 문제 데이터 구조 (평탄, 서버 연동 친화)
```
MathProblem {
  id, subject, chapter, lesson, difficulty,
  question, choices[4], answerIndex,
  explanation, detailedExplanation?, estimatedTime
}
```
`breadcrumb` = "수학Ⅰ > 수열 > 등차수열" 자동 생성. `fromJson/toJson` 구비.

### 4.2 상태 (Riverpod)
- `allProblemsProvider` — 전체 문제(현재 더미). **서버 연동 시 이 provider만 교체.**
- `statsProvider` (StatsNotifier) — 푼 수/정답수/연속/주간, `wrongIds`(오답 시 추가·정답 시 제거), 과목 진행률, 최근 기록, 이어풀기.
- `wrongProblemsProvider` — 다시 풀 문제 목록.
- `settingsProvider` — 알림/하루목표/테마(테마는 실제 앱에 적용).

통계 정의: 정답률 = 정답/푼 수, 진행률은 표시용 더미(추후 실제 집계로 교체).

## 6. 디자인 토큰

- 배경: 아주 연한 블루그레이(라이트) / 딥 그레이(다크)
- 포인트: 차분한 블루(#3A6FB0) + 민트(#2FB6A3)
- 정답: 부드러운 초록(#2E9E6B) / 오답: 부드러운 빨강(#D66A5F)
- 카드: radius 20, 약한 그림자, 옅은 외곽선
- 난이도: 단계별 6색(쉬움 블루 → 어려움 레드)

## 7. 로드맵

> 원칙: **기능을 먼저 단단히 완성한 뒤, 콘텐츠(2.3 목표 규모)를 점진 충전한다.**

**1단계 — 기능 강화 (먼저)**
- **영속화**: `StatsNotifier`/`SettingsNotifier`에 shared_preferences 로드/저장 (진도·설정 유지).
- 풀이 경험: 단답형 문제 타입, 풀이 중 힌트, 오답 노트/복습 큐(간격 반복).
- 진단/추천: 홈에 카드 추가로 합류(기존 흐름과 독립).
- 학습 리마인더 알림.

**2단계 — 콘텐츠 충전 (기능 안정화 후)**
- 목표: 소단원 × 난이도당 ~50문제 (2.3 참조), 단원/난이도 단위로 워크플로우 반복.
- 우선순위: 사용자 진입 많은 단원부터(중학·수Ⅰ 등) 난이도 골고루.

**3단계 — 확장**
- 서버 연동(`ContentRepository`의 index/과목 로드 메서드를 원격 호출로 교체, `MathProblem.fromJson` 이미 구비), 문제팩 다운로드.
- 광고 / 인앱결제(세션·탭 진입점 분리되어 삽입 지점 명확).
